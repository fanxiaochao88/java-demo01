# 文档二：你的代码跟企业级开发有多大差距？

## 先说结论

**你的代码组织方式是对的，方向完全正确。** 三层架构、统一响应封装、接口与实现分离，这些都是企业真实项目在用的模式。差距主要在于缺少了一些"防护网"。

---

## 对比表：你的项目 vs 企业级项目

| 维度 | 你的项目（现状） | 企业级项目 |
|------|--------------|---------|
| **目录结构** | `controller / service / mapper / pojo` ✅ | 相同，可能还有 `exception / config / utils / dto / vo` |
| **三层架构** | 完整 ✅ | 完整，职责划分更严格 |
| **统一响应格式** | `Result` 类（code/msg/data）✅ | 基本相同，可能加 timestamp、traceId |
| **依赖注入方式** | 构造器注入（`@RequiredArgsConstructor`）✅ 这是推荐方式 | 相同，大厂代码规范通常要求构造器注入 |
| **Lombok 使用** | `@Data / @NoArgsConstructor` ✅ | 相同 |
| **下划线转驼峰** | `map-underscore-to-camel-case: true` ✅ | 相同 |
| **异常处理** | ❌ 完全没有 | 必须有 `GlobalExceptionHandler` |
| **参数校验** | ❌ 完全没有（比如 name 传空字符串，直接插库） | 用 `@Valid + @NotBlank` 等注解校验 |
| **日志** | ❌ 没有 `log.info()` | 每个关键操作都要打日志，便于排查问题 |
| **事务管理** | ❌ 没有 `@Transactional` | 涉及多条 SQL 的操作必须加事务 |
| **SQL 方式** | 全注解（简单场景 ✅） | 复杂查询用 XML，简单 CRUD 用注解 |
| **数据分页** | ❌ 没有，`findAll` 直接全查 | 用 `PageHelper` 或 MyBatis Plus 分页 |
| **时间字段自动填充** | 在 Service 里手动 set ⚠️（能用，但重复代码多） | 用 MyBatis Plus 的 `@TableField(fill = ...)` 自动填充 |
| **配置文件** | 明文密码 `password: 123456` ⚠️ | 生产环境从环境变量或配置中心读取 |
| **接口文档** | ❌ 没有 | 用 Swagger/OpenAPI 自动生成文档 |
| **单元测试** | ❌ 只有一个空测试 | 每个 Service 方法都要有测试用例 |

## 你做得好的地方（认真说）

### 1. 构造器注入是对的

```java
// 你的代码 ✅
@RequiredArgsConstructor
@Service
public class DeptServiceImpl implements DeptService {
    private final DeptMapper deptMapper;   // final + 构造器注入
}
```

很多教程和新手用 `@Autowired` 字段注入，但这种方式有隐患（无法在单元测试中替换依赖）。
你用的方式才是现在 Spring 官方推荐的写法。

### 2. Service 接口 + 实现类分离是对的

```java
DeptService（接口）
    └── DeptServiceImpl（实现类）
```

这样做的好处：以后换实现（比如换一套缓存逻辑）不需要改 Controller。

### 3. `Result` 统一封装是对的

前端拿到的永远是 `{code, msg, data}` 这个结构，不用猜返回格式。

## 离企业级还差什么（优先级排序）

```
第 1 位：全局异常处理器   ← 最重要，没有这个线上出错就是一场噩梦
第 2 位：参数校验         ← 防止脏数据进库
第 3 位：分页查询         ← 生产数据量大，全查会崩
第 4 位：日志             ← 排查问题全靠它
第 5 位：事务             ← 多条 SQL 操作时保证数据一致性
第 6 位：单元测试         ← 保证代码质量
```

**你目前的阶段：打好了地基，把墙建起来了，但还没装防盗门和监控摄像头。**

---

## 一句话总结

你的代码结构和企业级开发是同一个套路，只是缺少了"防护层"。这很正常，这些防护层（异常处理、校验、事务、日志）本来就是后面几个阶段要学的内容。继续往下学，很快就会补上。
