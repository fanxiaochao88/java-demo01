## 一、框架搭建细节

### 1. Spring Boot 项目的 3 个"必须知道"的文件

```
pom.xml          ← Maven 依赖配置，告诉项目用哪些库
application.yaml ← 应用配置，比如数据库地址、端口
启动类.java       ← 程序入口，带 @SpringBootApplication 注解
```

### 2. pom.xml 关键部分

```xml
<!-- 父 POM：Spring Boot 的"大礼包"，帮你管理所有依赖版本，不用自己写版本号 -->
<parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>4.0.6</version>
</parent>

<!-- Web 开发必须的：包含 Spring MVC、Tomcat（内嵌服务器）等 -->
<dependency>
    <artifactId>spring-boot-starter-webmvc</artifactId>
</dependency>

<!-- MyBatis + Spring Boot 整合包 -->
<dependency>
    <artifactId>mybatis-spring-boot-starter</artifactId>
    <version>4.0.1</version>
</dependency>

<!-- MySQL 驱动，runtime 表示只在运行时需要，编译时不需要 -->
<dependency>
    <artifactId>mysql-connector-j</artifactId>
    <scope>runtime</scope>
</dependency>

<!-- Lombok：代码生成工具，optional 表示不会传递给依赖本项目的其他项目 -->
<dependency>
    <artifactId>lombok</artifactId>
    <optional>true</optional>
</dependency>
```

### 3. application.yaml 配置

```yaml
spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver   # MySQL 8+ 的驱动类名
    url: jdbc:mysql://localhost:3306/tlias          # 数据库连接地址
    username: root
    password: 123456

mybatis:
  configuration:
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl  # 在控制台打印 SQL
    map-underscore-to-camel-case: true  # create_time → createTime 自动转换
```

## 二、目录结构

```
com.itheima/
├── TliasWebManagementApplication.java   ← 启动类，必须在最外层（是所有子包的父包）
├── controller/     ← 接收 HTTP 请求，返回响应。不写业务逻辑
├── service/        ← 业务逻辑层
│   └── impl/       ← 接口的实现类放这里
├── mapper/         ← 数据库操作层（MyBatis 接口）
└── pojo/           ← Plain Old Java Object，存放实体类、VO、DTO 等数据类
```

**重要规则：** 启动类必须在 `com.itheima` 这个根包下，这样 Spring Boot 才能扫描到下面所有的 `@Component`、`@Service`、`@RestController` 等注解。

## 三、注解大全（你项目中出现的所有注解）

### Spring Boot / Spring MVC 注解

| 注解 | 放在哪里 | 作用 |
|------|---------|------|
| `@SpringBootApplication` | 启动类 | 等于 `@SpringBootConfiguration + @EnableAutoConfiguration + @ComponentScan` 三合一 |
| `@RestController` | Controller 类 | 等于 `@Controller + @ResponseBody`，让方法返回值直接转成 JSON |
| `@RequestMapping` | 类或方法 | 映射 URL 路径 |
| `@GetMapping` | 方法 | 处理 GET 请求，等于 `@RequestMapping(method=GET)` |
| `@PostMapping` | 方法 | 处理 POST 请求 |
| `@DeleteMapping` | 方法 | 处理 DELETE 请求 |
| `@PutMapping` | 方法 | 处理 PUT 请求 |
| `@RequestParam` | 方法参数 | 接收 URL 查询参数，如 `?id=5` |
| `@PathVariable` | 方法参数 | 接收路径参数，如 `/depts/{id}` |
| `@RequestBody` | 方法参数 | 接收请求体中的 JSON，自动反序列化成 Java 对象 |
| `@Service` | Service 实现类 | 告诉 Spring 这是一个 Bean，纳入 IoC 容器管理 |

### MyBatis 注解

| 注解 | 作用 | 例子 |
|------|------|------|
| `@Mapper` | 告诉 MyBatis 这是一个 Mapper 接口，自动生成实现类 | `@Mapper public interface DeptMapper` |
| `@Select` | 写查询 SQL | `@Select("select * from dept")` |
| `@Insert` | 写插入 SQL | `@Insert("insert into dept ...")` |
| `@Update` | 写更新 SQL | `@Update("update dept set ...")` |
| `@Delete` | 写删除 SQL | `@Delete("delete from dept where id=#{id}")` |

### Lombok 注解

| 注解 | 作用 |
|------|------|
| `@Data` | 自动生成 getter、setter、toString、equals、hashCode |
| `@NoArgsConstructor` | 自动生成无参构造器 |
| `@AllArgsConstructor` | 自动生成全参构造器 |
| `@RequiredArgsConstructor` | 为所有 `final` 字段生成构造器（用于构造器注入） |

## 四、三层架构的职责划分

```
HTTP 请求
    ↓
Controller（控制层）
    职责：接收参数、调用 Service、返回 Result
    不能写业务逻辑！不能直接调 Mapper！
    ↓
Service（业务层）
    职责：写业务逻辑（比如 add 时补全 createTime）
    不能直接写 SQL！
    ↓
Mapper（数据层）
    职责：只负责和数据库交互，写 SQL
    不能写业务逻辑！
```

## 五、参数传递方式

### 1. URL 查询参数（Query String）

```
请求：DELETE /depts?id=5
                    ↑ 问号后面是查询参数

接收方式：
@DeleteMapping("/depts")
public Result delete(@RequestParam Integer id) { ... }
//                   ↑ 用 @RequestParam 接收
```

### 2. 路径参数（Path Variable）

```
请求：GET /depts/5
                ↑ 5 是路径的一部分

接收方式：
@GetMapping("/depts/{id}")
//                  ↑ 用 {} 占位
public Result getById(@PathVariable Integer id) { ... }
//                     ↑ 用 @PathVariable 接收
```

### 3. 请求体参数（Request Body，JSON）

```
请求：POST /depts
      请求体：{"name": "研发部"}

接收方式：
@PostMapping("/depts")
public Result add(@RequestBody Dept dept) { ... }
//                 ↑ 用 @RequestBody 接收，Spring 自动把 JSON 转成 Dept 对象
```

## 六、MyBatis 参数占位符 `#{}`

```java
@Select("select * from dept where id = #{id}")
//                                    ↑ #{} 是安全的占位符，防 SQL 注入
//                                      会被替换成 PreparedStatement 的 ?
Dept findById(Integer id);

@Insert("insert into dept(name, create_time) values(#{name}, #{createTime})")
//                                                    ↑ 对象的字段名，注意是驼峰
void add(Dept dept);
```

**记住：用 `#{}` 不用 `${}`，`${}` 不安全，有 SQL 注入风险。**

## 七、IoC 容器和依赖注入

**概念：** Spring 的 IoC 容器负责创建和管理对象（Bean）。你不用 `new DeptMapper()`，Spring 自动帮你创建并注入。

```java
// 方式一：字段注入（简单但不推荐）
@Autowired
private DeptMapper deptMapper;  // Spring 直接塞进来

// 方式二：构造器注入（你的项目用的，推荐）
@RequiredArgsConstructor
public class DeptServiceImpl {
    private final DeptMapper deptMapper;  // final + 构造器，Spring 通过构造器传入
}
```

## 八、统一响应封装 Result

```java
// 三个字段
code = 1   // 成功
code = 0   // 失败
msg        // 错误信息（成功时可以为 "success" 或 null）
data       // 业务数据（失败时为 null）

// 使用方式
return Result.success();           // 操作成功，无数据
return Result.success(list);       // 查询成功，带数据
return Result.error("删除失败");    // 失败，带错误信息
```
