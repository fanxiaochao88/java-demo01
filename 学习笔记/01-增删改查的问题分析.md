# 文档一：增删改查的问题分析 —— 为什么感觉"默认全是成功"

## 你的感觉是对的

现在的代码确实有问题。让我们先看看你的 Mapper 接口：

```java
void deleteById(Integer id);   // 返回 void，没有任何返回值
void add(Dept dept);           // 返回 void
void update(Dept dept);        // 返回 void
```

再看 Controller 层：

```java
// DeptController.java
public Result delete(@RequestParam Integer id) {
    deptService.deleteById(id);
    return Result.success();    // ← 不管数据库实际发生了什么，永远返回成功
}
```

**问题在于：不管数据库有没有真正执行成功，Controller 都直接 `return Result.success()`。**

---

## 举几个会"静默失败"的场景

| 场景 | 实际发生了什么 | 你的接口返回什么 |
|------|--------------|----------------|
| 删除一个不存在的 id（如 id=9999） | 0 条数据被删除 | `{"code":1,"msg":"success"}` ← 谎称成功 |
| 修改一个不存在的 id | 0 条数据被修改 | `{"code":1,"msg":"success"}` ← 谎称成功 |
| 数据库连接断开 | SQL 根本没执行 | 这个会抛异常，但没有被捕获，会返回 500 错误 |
| 插入时字段不合法（超长） | 数据库报错 | 同上，会抛异常，返回 500 |

---

## 如何改？—— 两个层面

### 第一层：让 Mapper 返回影响的行数

MyBatis 的增删改操作可以返回 `int`，表示"影响了几行"：

```java
// 修改前（现在的代码）
void deleteById(Integer id);

// 修改后（能知道有没有真的删到数据）
int deleteById(Integer id);   // 返回影响的行数，0 表示没删到，1 表示删到了
int add(Dept dept);
int update(Dept dept);
```

### 第二层：在 Service 或 Controller 里判断结果

```java
// Controller 里判断
public Result delete(@RequestParam Integer id) {
    int rows = deptService.deleteById(id);
    if (rows > 0) {
        return Result.success();
    } else {
        return Result.error("删除失败，该部门不存在");
    }
}
```

### 第三层（进阶）：全局异常处理器

上面处理的是"没找到数据"的情况。但如果数据库崩了、SQL 写错了，会抛出 Exception，
现在你的程序没有任何处理，Spring Boot 会直接返回一个 HTML 格式的 500 错误页，前端根本解析不了。

正确的做法是加一个**全局异常处理器**：

```java
@RestControllerAdvice   // 这个注解让这个类能捕获所有 Controller 抛出的异常
public class GlobalExceptionHandler {

    @ExceptionHandler(Exception.class)  // 捕获所有 Exception
    public Result handleException(Exception e) {
        // 打印日志（生产环境必须有）
        e.printStackTrace();
        // 给前端返回统一格式的错误信息
        return Result.error("操作失败：" + e.getMessage());
    }
}
```

把这个类放在 `controller` 包下，Spring Boot 会自动发现它。

---

## 总结

| 问题 | 现状 | 改进方向 |
|------|------|---------|
| 删除/修改不存在的数据 | 谎称成功 | Mapper 返回 `int`，判断影响行数 |
| 数据库异常（连接断、SQL 错） | 返回 500 HTML 页面，前端懵 | 加 `GlobalExceptionHandler` |
| 完全没有失败路径 | `Result.error()` 方法写了但从没被调用过 | 上面两步改完就会用到了 |

**你的 `Result.error()` 方法目前是摆设，从来没有被调用过。** 这是一个典型的入门项目问题，很正常，继续学下去会专门讲这个。
